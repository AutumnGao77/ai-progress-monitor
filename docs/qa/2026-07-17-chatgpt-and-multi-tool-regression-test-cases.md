## 2. 监控能力边界

验收时必须按下表判断，不能要求进程级工具提供代码中不存在的完整状态，也不能接受深度监控工具长期只显示空闲入口。

| 类型 | 工具 | 当前能力 | 正确预期 |
|---|---|---|---|
| 深度桌面监控 | ChatGPT Desktop | 读取明确标记为桌面来源的本地会话事件 | 能显示具体会话、进行中、完成待查看、等待输入、点击跳转和 15 分钟收口 |
| 深度桌面监控 | Qoder / Qoder CN | App 进程 + 日志 + 本地缓存 | 能显示真实标题、多个任务、三态、用户注意状态和点击跳转 |
| 深度桌面监控 | WorkBuddy | App 进程 + 本地会话库 + runtime log | 能显示具体会话、项目上下文、三态、用户注意状态和点击跳转 |
| 增强 CLI 监控 | Claude Code CLI | CLI 进程 + Claude 本地会话状态 | 能识别运行与完成待查看；无法读取时保守降级为进程级状态 |
| 进程级 CLI | Codex、ChatGPT CLI、Qoder、Qoder CN、WorkBuddy、CodeBuddy、Gemini、Gemini CLI、Cursor Agent、Qwen Code、Aider、OpenCode、Goose、Continue、Kiro、Perplexity、Poe | 只确认进程、活跃度、父 GUI 和 cwd | 可见、可聚焦、退出后消失；活跃/静默判断允许保守，不得伪报完整进度或终端内容 |
| 进程级桌面 App | Claude、Gemini、Perplexity、Poe、Kiro | 只确认桌面主进程存在 | 无具体数据源时显示空闲入口；点击能激活 App；不得仅凭 App 存活显示进行中或待处理 |
| 可靠桥接 | JSON 事件源与 wrapper 脚本 | 由外部事件明确上报会话状态 | 对任意已配置工具可形成完整会话；完整项应压过同进程的重复进程级气泡 |

## 3. 状态判定口径

| 场景 | 气泡状态 | 点击后的正确行为 |
|---|---|---|
| AI 正在工作 | 进行中，绿色 | 保持进行中 |
| AI 已返回结果，只等用户查看 | 待处理，红色 | 聚焦成功后转为空闲 |
| 等待用户批准、输入、选择或继续 | 待处理，红色 | 聚焦成功后仍保持待处理，直到工具内状态真实变化 |
| 无具体会话，仅 App 存活 | 空闲，蓝色 | 保持空闲并激活 App |
| 通用 CLI 活跃 | 进行中或保守状态 | 点击回到对应终端/IDE；不得声称读到了完整进度 |
| 通用 CLI 静默 | 空闲 | 点击回到对应终端/IDE |

## 16. 放行标准

| 等级 | 标准 |
|---|---|
| P0 放行门 | 所有适用 P0 用例通过；ChatGPT 状态和跳转无回归；其他工具不会消失、串状态、串跳转或重复计数 |
| P1 放行门 | 本机具备条件的 P1 用例通过；无条件项记录 `N/A` 和原因 |
| 自动化门 | 全量单元测试通过；macOS 原生 App 编译和签名检查通过；严格手测日志检查通过 |
| 隐私门 | 气泡、通知、日志、截图中没有 prompt、回复、命令输出、完整真实路径、内部 ID 或敏感信息 |
| 打包门 | 用户明确确认手测通过后，才能执行发布包构建 |

以下任一情况出现，都必须判定本轮不通过：

- ChatGPT 任务运行时仍长期只有空闲入口；
- 点击 ChatGPT 气泡仍提示“无法定位窗口”；
- 桌面列表重新出现 `Codex Desktop`；
- Codex CLI 被误标成 ChatGPT Desktop；
- Qoder、Qoder CN 或 WorkBuddy 有具体会话时只显示 App 空闲入口；
- 真正等待批准/输入的待处理气泡被点击后错误清空；
- 一个来源异常导致全部工具消失或 Pet 停止刷新；
- 其他 App / CLI 因本次修复不再出现、无法跳转或退出后永久残留；
- 严格手测检查仍存在 `[TODO]`；
- 未经用户确认就打包。
