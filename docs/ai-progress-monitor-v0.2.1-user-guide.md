# AI Progress Monitor v0.2.1 使用说明

AI Progress Monitor 是一个桌面小助手：把正在使用的 AI 工具状态收在桌面角落。它不会替你回答问题；当 AI 正在工作、需要你查看或暂时空闲时，树懒 Pet 会用角标和气泡告诉你，点一下就能回到对应窗口继续处理。

| 你的情况 | 下载内容 | 下载链接 | 打开后有什么 |
|---|---|---|---|
| 使用 Apple 芯片 Mac（M1 / M2 / M3 / M4），只想直接使用 | macOS 用户包 | [下载 macOS 版](https://github.com/AutumnGao77/ai-progress-monitor/releases/download/v0.2.1/AI-Progress-Monitor-v0.2.1-macOS-arm64.zip) | `AI Progress Monitor.app`、`README.txt`、`LICENSE` |

当前正式版仅支持 macOS。请下载 macOS 用户包；无需下载或使用其他安装包。

## 使用前请确认

| 项目 | 要求或说明 |
|---|---|
| 已稳定验收的平台 | macOS 13 或更高版本，且为 Apple 芯片 Mac |
| 运行环境 | 需要 Python 3.9 或更高版本 |
| 支持范围 | 当前正式版仅支持 macOS |
| AI 工具 | 可识别 ChatGPT 桌面端、Claude Code、Codex CLI、Qoder / Qoder CN、WorkBuddy 等已配置工具；不同工具的状态精度会有差异 |
| 隐私 | 监控在本机完成；Pet 默认不显示你的提示词或回答正文 |

## 3 分钟上手

1. 下载并解压 macOS 包。
2. 双击 `AI Progress Monitor.app`。
3. 桌面右下角出现树懒 Pet，说明监控已开始。
4. 像平时一样使用 ChatGPT、Claude Code、Codex 或其他已支持工具。
5. 看到 Pet 右上角出现数字时，左键点击 Pet 查看气泡；再点击某条气泡，APP 会尝试带你回到对应的 AI 工具窗口。

首次打开时，macOS 可能提示无法验证开发者。这是因为 v0.2.1 使用本地 ad-hoc 签名，尚未经过 Apple 公证。请右键点击 App 选择“打开”，或到“系统设置 → 隐私与安全性”中允许打开；不需要关闭整个系统的安全保护。

## 你会看到什么

Pet 平时安静待在桌面角落，不会弹出一个占满屏幕的控制面板。

| 看到的内容 | 表示什么 | 你可以怎么做 |
|---|---|---|
| 红色角标 / 待处理 | AI 已给出结果，或正在等待你处理 | 点击 Pet，再点击对应气泡回到原窗口 |
| 绿色角标 / 进行中 | AI 仍在工作 | 继续手头的事情，稍后再看 |
| 蓝色角标 / 空闲 | 已识别到工具，但当前没有需要处理的任务 | 无需操作 |
| 气泡中的工具名、项目文件夹或对话标题 | 帮你区分是哪一个任务 | 点击气泡定位到对应工具 |

角标数字是当前被监控会话的总数，不只是“待处理”的数量。多个状态同时存在时，颜色优先提示最需要关注的状态。

## 常用操作

| 你要做什么 | 怎么操作 | 会发生什么 |
|---|---|---|
| 查看所有会话 | 左键点击 Pet | Pet 上方展开气泡列表；再次点击会收起 |
| 回到某个 AI 对话 | 点击该对话的气泡 | 尝试聚焦对应 AI 工具窗口；成功查看后，部分待处理会话会转为空闲 |
| 移动 Pet | 按住 Pet 拖到合适位置 | 松开后保存位置；不会误触发展开列表 |
| 换外观 | 右键 Pet → `外观` → 选择“背带裤树懒”或“衬衫树懒” | 当前选择显示对勾，并在下次启动后保留 |
| 暂时不看 Pet | 右键 Pet → `隐藏 Pet` | Pet 消失，但监控继续运行；可从菜单栏头像图标恢复显示 |
| 完全关闭 | 右键 Pet → `退出程序` | 关闭 Pet 和本地监控服务 |

## 它适合做什么

- 同时开了几个 AI 任务，不想频繁切窗口确认进度。
- 希望在 AI 完成或需要处理时得到一个轻量提醒。
- 想从桌面快速跳回某个 ChatGPT、CLI 或 AI 编辑器窗口。
- 想知道哪些 AI 工具还开着，而不想让监控器读取或展示对话内容。

## 使用限制

| 限制 | 请这样理解 |
|---|---|
| Pet 不是聊天窗口 | 不提供直接回复、批准命令或阅读长回答的入口；请点击气泡回到原工具处理 |
| 打开前的旧历史不会全部出现 | APP 从启动后开始监控，避免一打开就堆满旧任务；但仍在运行的 CLI 或桌面 App 可能显示为空闲入口 |
| 状态不是每个工具都同样精确 | ChatGPT 桌面端和 Claude Code 可识别更多会话信号；Codex、Qoder、WorkBuddy 等 CLI 有时只能根据进程活动做保守判断 |
| 窗口跳转可能受系统权限影响 | 有辅助功能权限时定位更准确；没有权限时会尽量激活对应 App。若失败，手动切回原窗口即可 |
| 已查看的桌面对话不会永久保留 | 转为空闲后会保留约 15 分钟，随后从列表移出；对应桌面 App 仍开着时，会留下一个空闲入口 |
| 当前不支持 Intel Mac | macOS 用户包只面向 Apple Silicon；Intel Mac 不在 v0.2.1 的已验收范围内 |
