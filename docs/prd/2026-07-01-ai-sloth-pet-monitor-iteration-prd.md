# 原创树懒 Pet 监控助手迭代 PRD

## 1. 结论

本次迭代建议将 AI Progress Monitor 从“工具窗口式监控器”升级为 **原创树懒桌面 Pet**：默认以动画人物常驻桌面，通过树懒动作、气泡列表和右上角数字角标表达 Claude Code、Codex 及后续通过工具定义表配置的 AI 工具并行工作状态。

新版主体验只保留三件事：**看状态、看数量、点气泡回到对应窗口**。不再把诊断、隐藏、暂停、暂隐、详情页、宠物内 Yes/No 操作放在用户主路径里。

## 2. 背景与问题

当前版本已经具备悬浮监控能力，但界面仍偏“工具窗口”，对用户来说信息和按钮过多。用户真正需要的是快速知道：哪个 AI 对话在跑、哪个需要处理、点哪里能回到现场。

| 当前问题 | 用户影响 | 新版方向 |
|---|---|---|
| 点击展开后内容过多 | 用户需要阅读和判断，增加负担 | 改为 Pet 上方气泡列表 |
| 诊断、已隐藏、暂停、暂隐等按钮外露 | 主路径变复杂，像管理工具 | 从主体验移除 |
| 现有 Pet 更像小窗口 | 缺少角色感和状态表达 | 改为原创树懒动画人物 |
| 只显示文件夹名会有歧义 | 同文件夹多个对话无法区分 | 增加工具/对话标识 |
| 状态信息可能暴露对话内容 | 不利于隐私与低干扰 | 只展示文件夹、对话标识和状态 |

## 3. 产品目标

| 目标 | 说明 |
|---|---|
| 降低信息负担 | 默认不展示完整列表、详情页或对话内容 |
| 强化待处理提醒 | 有待处理对话时，Pet 动作和角标明显变化 |
| 支持并行任务识别 | 多个 Claude Code / Codex 对话同时运行时可区分 |
| 快速回到工作现场 | 点击气泡即可跳转到对应窗口或页面 |
| 提升情绪体验 | 用原创树懒角色承载状态，减少工具感 |
| 降低公开发布风险 | 不使用受版权保护的电影角色名称、造型或可识别形象 |

## 4. 目标用户

主要用户是同时使用 Claude Code、Codex 等 AI 编程工具的产品、研发或文档工作者。用户可能同时开多个项目、多个终端、多个桌面端 AI 对话或多个 Codex 页面，需要一个低干扰入口帮助自己掌握并行任务状态。

## 5. 核心体验

| 用户动作 | 产品响应 |
|---|---|
| 默认不操作 | 原创树懒 Pet 常驻桌面，右上角显示角标 |
| 点击 Pet | 展开或收起 Pet 上方气泡列表 |
| 右键点击 Pet | 展示轻量菜单：隐藏 Pet、退出程序 |
| 查看气泡 | 每条气泡展示文件夹名、对话标识和状态 |
| 点击气泡 | 跳转到对应 Claude Code / Codex / 已配置 AI 工具窗口或页面 |
| 有待处理对话 | Pet 动作变明显，待处理气泡优先展示，角标高亮 |
| 无待处理对话 | Pet 保持低干扰动作，不主动打断用户 |

## 6. 功能需求

### 6.1 原创树懒 Pet

| 要求 | 说明 |
|---|---|
| 角色定位 | 原创树懒助手，气质慢、温和、可靠，有轻微反差感 |
| 合规边界 | 不使用《疯狂动物城》角色名称、造型、服装或可识别表情设计 |
| 默认形态 | 小尺寸常驻桌面，不遮挡主要工作区域 |
| 资源形态 | 本地内置三态 PNG 图片，空闲、进行中、待处理分别切换；支持偏好文件覆盖本地图片路径 |
| 动画表达 | 根据状态变化切换动作，而不是依赖大段文字提醒 |
| 点击行为 | 点击 Pet 只负责展开/收起气泡列表 |
| 右键行为 | 右键 Pet 展示隐藏 Pet、退出程序两个选项 |

### 6.2 用户可见状态

用户侧只展示三种状态，避免把底层技术状态全部暴露给用户。

| 状态 | 用户含义 | 树懒动作 | 优先级 |
|---|---|---|---|
| 待处理 | 需要用户回到对应对话处理 | 抬头、举小牌、轻微挥手，动作更明显 | 高 |
| 进行中 | AI 正在执行、思考、运行命令或生成内容 | 慢速敲键盘、眨眼、缓慢点头 | 中 |
| 空闲 | 当前对话暂无动作，可能已完成或等待下一步 | 打盹、放松、慢慢缩回 | 低 |

### 6.3 气泡列表

| 要求 | 说明 |
|---|---|
| 展示位置 | 气泡列表显示在 Pet 上方 |
| 展开方式 | 点击 Pet 展开，再次点击 Pet 收起 |
| 信息内容 | 每条气泡只展示文件夹名、对话标识和状态 |
| 隐私边界 | 不展示具体对话内容、命令输出、用户输入或完整提示词 |
| 排序规则 | 待处理优先，其次进行中，最后空闲；同状态内按更新时间倒序 |
| 点击行为 | 点击某条气泡跳转到对应窗口或页面 |

### 6.4 监控起点与桌面对话收口

| 规则 | 要求 |
|---|---|
| App 启动前历史文件会话 | 不回捞为本轮气泡列表，避免一启动出现大量旧待处理 |
| App 启动前已存在且当前仍存活的 CLI | 属于现场，可显示为空闲或当前状态；后续活动继续更新 |
| App 启动前已打开且当前仍存活的桌面 App | 可显示 App 空闲入口；不能仅因主程序存活显示进行中或待处理 |
| 具体桌面对话出现 | 具体对话优先展示，并隐藏同工具的通用 App 空闲入口，避免重复计数 |
| 桌面端具体对话待处理 | 必须保留到用户点击查看/处理，不能因为时间过长自动消失 |
| 已查看后的桌面端具体对话 | 转为空闲后在气泡列表保留 15 分钟；超过 15 分钟自动移出 |
| 桌面端具体对话都被移出且 App 仍存活 | 保留该工具的 App 空闲入口 |

### 6.5 收起态数字角标

| 场景 | 角标规则 | 颜色 |
|---|---|---|
| 无对话 | 不显示角标，或显示弱提示状态 | 无 |
| 有待处理对话 | 显示气泡列表总数 | 红色 |
| 无待处理但有进行中对话 | 显示气泡列表总数 | 绿色 |
| 仅有空闲对话 | 显示空闲对话数量 | 蓝色 |
| 气泡展开时 | 角标可保留，但视觉优先级低于气泡内容 | 跟随当前最高优先级状态 |

角标数字显示当前气泡列表中的会话总数；角标颜色按待处理 > 进行中 > 空闲显示最高优先级状态。例如 1 个待处理、2 个进行中时，角标显示 `3` 且为红色。

### 6.6 同文件夹多对话区分

同一个文件夹下可能同时运行多个 Claude Code 或 Codex 对话，气泡不能只显示文件夹名，否则用户无法判断点击后会回到哪个对话。

| 场景 | 气泡显示规则 | 示例 |
|---|---|---|
| 单文件夹单对话 | `文件夹名 · 状态` | `checkout-flow · 进行中` |
| 单文件夹多对话 | `文件夹名 · 工具 + 序号 · 状态` | `checkout-flow · Codex #2 · 待处理` |
| 有明确会话别名 | `文件夹名 · 会话别名 · 状态` | `checkout-flow · 支付文案 · 进行中` |
| 有可用窗口标题 | 提取安全短标题，不展示正文 | `checkout-flow · PRD polish · 空闲` |
| 无法识别标题 | 使用稳定序号或短 ID | `checkout-flow · Claude #1 · 进行中` |
| 桌面端无真实项目文件夹 | 使用工具名 + 安全短标题或“工具名 对话” | `Codex · hello · 空闲`、`Codex 对话 #1 · 空闲` |
| 工具定义表声明的自动聊天目录 | 视为无真实项目文件夹，不当成项目名展示 | `Codex · hello · 空闲` |

气泡标识优先级：

| 优先级 | 标识来源 | 使用规则 |
|---|---|---|
| 1 | 用户自定义别名 | 如果已有别名，优先展示 |
| 2 | 安全窗口标题摘要 | 只提取短标题，不展示对话正文 |
| 3 | 工具名 + 稳定序号 | 同文件夹多对话默认兜底 |
| 4 | 短 ID | 标题和工具信息都不足时使用 |

稳定序号需在同一运行周期内保持不变，避免刷新后同一对话编号频繁跳动。

自动聊天目录不是 Codex 特例。后续接入其他 AI 桌面端时，应通过工具定义表配置 `key`、`display_name`、桌面主程序路径、CLI 可执行名和自动聊天目录 pattern，前端不硬编码具体产品路径。

### 6.7 点击跳转

| 要求 | 说明 |
|---|---|
| 点击对象 | 用户点击具体气泡，而不是点击 Pet 本体跳转 |
| 跳转目标 | 对应 Claude Code / Codex / 已配置 AI 工具的终端窗口、桌面窗口或页面 |
| 跳转失败 | 不在主界面展开诊断，只给轻量失败提示 |
| 跳转依据 | 优先使用 `window_id` / `process_id` / `focus_process_id` / `focus_app_name`，失败时使用窗口标题或项目路径兜底 |

### 6.8 右键菜单与系统托盘

| 菜单项 | 行为 | 说明 |
|---|---|---|
| 隐藏 Pet | 隐藏桌面 Pet，不关闭监控程序 | 程序继续在后台运行，继续监控状态 |
| 退出程序 | 退出桌面 Pet 和本地监控服务 | 属于关闭应用，不再继续监控 |

隐藏 Pet 后，用户需要能通过电脑状态栏小图标再次唤醒：

| 系统 | 唤醒入口 | 要求 |
|---|---|---|
| macOS | 菜单栏状态图标 | 点击或菜单选择后重新显示 Pet |
| Windows | 系统托盘图标 | 双击或菜单选择后重新显示 Pet |

“隐藏 Pet”只影响桌面可见性，不等同于“隐藏某个会话”。新版主体验不提供单条会话隐藏管理。

## 7. 接口与数据影响

本次 PRD 不要求新增对外 API。实现时优先复用现有数据字段生成气泡内容和跳转目标。

| 现有字段 | 用途 |
|---|---|
| `title` | 提取文件夹名、窗口短标题或对话标识 |
| `session_id` | 生成稳定短 ID 或兜底标识 |
| `tool` | 区分 Claude Code / Codex |
| `status` | 映射为用户可见三态 |
| `window_id` | 优先用于跳转对应窗口 |
| `process_id` | 窗口 ID 不可用时用于跳转兜底 |
| `focus_process_id` | CLI 或 IDE 内置终端场景下，优先回到父 GUI 应用 |
| `focus_app_name` | 窗口 ID 不可用时用于回到对应桌面应用 |
| `tool_display_name` | 未进入固定枚举的 AI 工具显示名 |
| `generated_conversation_path` | 标记工具定义表识别出的自动聊天目录，前端按无真实项目文件夹处理 |

状态映射规则：

| 底层状态 | 用户展示 |
|---|---|
| `needs_action` | 待处理 |
| `running` | 进行中 |
| `idle` / `completed` | 空闲 |
| `stuck` / `unknown` | 不作为主状态展示，必要时归入弱提示或内部日志 |

## 8. 非目标

| 不做 | 原因 |
|---|---|
| 不做大详情页 | 避免回到工具窗口体验 |
| 不在主体验展示诊断 | 诊断属于排障能力，不是日常状态查看 |
| 不做已隐藏会话管理入口 | 减少用户理解成本 |
| 不做暂停、暂隐按钮 | 收起/展开由点击 Pet 自然完成；Pet 级隐藏放入右键菜单 |
| 不做宠物内 Yes/No 操作 | 统一引导用户回到对应窗口处理 |
| 不展示完整对话内容 | 保护隐私，降低干扰 |
| 不复刻电影角色 | 降低公开发布、GitHub 分享和后续商用风险 |

## 9. 核心用户场景

| 场景 | 用户状态 | 产品表现 |
|---|---|---|
| 多任务并行 | 用户在文档或浏览器中工作 | Pet 右上角显示总数，点击可查看气泡 |
| 同文件夹多个对话 | 同一项目下开了多个 Codex/Claude 对话 | 气泡用工具名、别名或序号区分 |
| 某个对话待处理 | 用户没有盯着原窗口 | Pet 动作变明显，角标用红色显示气泡列表总数 |
| 用户想回到现场 | 用户看到某条气泡 | 点击该气泡后跳转到对应窗口或页面 |
| 对话正在运行 | 用户只想知道是否还在工作 | 气泡显示进行中，Pet 保持低干扰动作 |
| 对话空闲 | 用户想确认任务是否结束 | 气泡显示空闲，Pet 打盹或放松 |
| 已查看桌面对话 | 用户点击气泡回到桌面端查看后没有继续操作 | 具体对话空闲气泡保留 15 分钟后移出；桌面 App 仍存活时显示 App 空闲入口 |
| 用户暂时不想看到 Pet | 用户右键点击 Pet 并选择隐藏 Pet | Pet 消失，监控继续运行，可从状态栏/托盘恢复 |
| 用户不再需要监控 | 用户右键点击 Pet 并选择退出程序 | Pet 和本地监控程序退出 |

## 10. 验收标准

| 指标 | 验收标准 |
|---|---|
| 状态及时性 | 对话进入待处理后 5 秒内在 Pet 和气泡中体现 |
| 认知效率 | 用户 3 秒内能判断是否有待处理对话 |
| 多对话展示 | 至少支持同时展示 3 个以上对话气泡 |
| 同文件夹区分 | 同一文件夹下多个对话不会只显示重复文件夹名 |
| 收起态提示 | 气泡列表收起时，Pet 右上角显示数字角标 |
| 角标颜色 | 待处理显示红色，进行中显示绿色，空闲显示蓝色 |
| 角标优先级 | 多种状态并存时，角标数字显示总数，颜色按待处理 > 进行中 > 空闲显示 |
| 点击跳转 | 点击气泡可回到对应窗口或页面 |
| 桌面对话收口 | 已查看后转为空闲的桌面端具体对话保留 15 分钟后自动移出；如果桌面 App 仍存活，则保留 App 空闲入口 |
| 右键菜单 | 右键 Pet 可看到隐藏 Pet、退出程序 |
| 隐藏恢复 | 选择隐藏 Pet 后程序继续运行，用户可从状态栏/托盘小图标恢复 |
| 退出程序 | 选择退出程序后 Pet 和本地监控服务关闭 |
| 信息脱敏 | 气泡不展示具体对话内容、命令输出或用户输入 |
| 低干扰 | 进行中和空闲状态不主动弹出大面板 |
| 合规边界 | PRD 和公开版本不要求复刻受版权保护角色形象 |

## 11. 风险与约束

| 风险 | 说明 | 应对 |
|---|---|---|
| 角色 IP 风险 | 直接使用电影角色形象可能影响公开发布 | 使用原创树懒助手，只保留慢节奏性格灵感 |
| 跳转失败 | 系统权限或窗口识别不稳定时可能无法激活窗口 | 优先窗口 ID / 进程 ID，失败时标题兜底并轻提示 |
| 同名文件夹 | 多个路径最后一级文件夹同名时可能混淆 | 后续可在气泡悬停或辅助信息中补充上级路径 |
| 标题含敏感信息 | 窗口标题可能包含任务内容 | 只提取安全短标题，必要时降级为工具名 + 序号 |
| 动效过度 | 动作太明显会打扰用户 | 仅待处理状态明显增强，其他状态保持轻量 |
| 透明背景退化 | 图片或 Web 容器带底色、阴影时，原生悬浮窗会像一块灰底贴片 | 三态 PNG 保持透明角，APP 头像保持无水印、无圆外方框背景的透明圆形，透明像素应为 `(0,0,0,0)`；`.pet` 不额外添加 `drop-shadow`，替换图片后做真实悬浮窗检查 |

## 12. 默认决策

| 项目 | 决策 |
|---|---|
| 文档方式 | 新建迭代 PRD，不覆盖旧 PRD |
| 角色策略 | 使用原创树懒助手，避免公开发布版权风险 |
| 主体验 | 只保留 Pet、气泡、角标、点击跳转 |
| 操作边界 | 不在 Pet 内处理 Yes/No 或其他确认 |
| Pet 隐藏 | 右键菜单提供 Pet 级隐藏，不提供会话级隐藏管理 |
| 状态范围 | 用户可见状态只保留待处理、进行中、空闲 |
| 工具接入 | 通过工具定义表配置 CLI、桌面 App 和自动聊天目录，后续接入新 AI 工具不改前端命名逻辑 |
| 桌面对话收口 | 已查看后转为空闲的桌面端具体对话保留 15 分钟后移出 |
| 同文件夹区分 | 默认使用工具名 + 稳定序号，有更好标题或别名时优先展示 |
| 视觉资源 | 已落地三态 Pet 图片和 APP 头像；后续换图优先使用 `pet_assets` 配置，不改角标、气泡和状态逻辑 |
